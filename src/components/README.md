# Компоненты авторизации

## AuthGuard
Компонент для защиты приватных маршрутов. Проверяет наличие и валидность JWT токена в localStorage.

**Использование:**
```jsx
<AuthGuard>
  <YourProtectedComponent />
</AuthGuard>
```

**Поведение:**
- Показывает загрузку во время проверки авторизации
- Если пользователь не авторизован, редиректит на `/login`
- Если пользователь авторизован, показывает содержимое

## PublicRoute
Компонент для публичных маршрутов (например, страница логина). Проверяет авторизацию и редиректит авторизованных пользователей.

**Использование:**
```jsx
<PublicRoute>
  <LoginPage />
</PublicRoute>
```

**Поведение:**
- Показывает загрузку во время проверки авторизации
- Если пользователь авторизован, редиректит на главную страницу
- Если пользователь не авторизован, показывает содержимое

## Архитектура авторизации

1. **Проверка токена**: Используется JWT токен из localStorage
2. **Валидация**: Проверяется срок действия токена
3. **Очистка**: Невалидные токены автоматически удаляются
4. **Редиректы**: Используется React Router Navigate вместо window.location
5. **UX**: Показывается загрузка во время проверки авторизации

## Best Practices

- ✅ Проверка авторизации происходит до загрузки страницы
- ✅ Нет показа ошибок "Пожалуйста, авторизуйтесь"
- ✅ Плавные редиректы без перезагрузки страницы
- ✅ Автоматическая очистка невалидных токенов
- ✅ Разделение логики для приватных и публичных маршрутов
